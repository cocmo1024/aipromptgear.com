# Operator Runbooks

The most durable prompt systems tend to support an operator rather than pretending no operator exists. A runbook structure makes ownership visible and keeps workflow design compatible with real teams.

## Core ingredients

- A clear trigger, such as a ticket, lead, incident, or research request.
- A bounded task sequence instead of a single oversized prompt.
- Review checkpoints where a person can edit, approve, or escalate.
- Logging that captures inputs, outputs, and the model or prompt version used.

## Failure modes to plan for

- Source material is incomplete or contradictory.
- The system answers confidently when it should ask for clarification.
- The human reviewer receives too much output to audit quickly.
