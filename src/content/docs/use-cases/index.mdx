# Team Scenarios

Start here when the real question is not "which model should we use?" but "which team is trying to improve which recurring work?" This section organizes prompting and agent systems around real operating contexts rather than generic AI enthusiasm.

<div class="signal-strip">
	<div>
		<strong>Best for</strong>
		<p>Support, revenue, research, and operations teams working on repeatable processes with visible review points.</p>
	</div>
	<div>
		<strong>Avoid this mistake</strong>
		<p>Do not jump straight into prompts or models until the workflow owner, source material, output format, and review boundary are clear.</p>
	</div>
	<div>
		<strong>What comes next</strong>
		<p>After the team scenario is framed, move into workflows, then models, then tooling and evaluation.</p>
	</div>
</div>

## Decision flow

<Steps>
	1. Define the team, the recurring task, and the output that must improve.
	2. Identify the sources of truth, review boundary, and the highest-cost failure modes.
	3. Decide where the system should assist, draft, route, or escalate.
	4. Move into workflow and model design only after those boundaries are written down.
</Steps>
