# Tooling

Tooling is where prompt systems become maintainable. The goal is not to collect every possible platform, but to choose the minimum stack that supports visibility, review, versioning, and reliable rollout.

## Tooling choices should answer

<Steps>
	1. Where are prompts stored and versioned?
	2. How are prompts connected to workflow versions and model versions?
	3. What traces or examples can reviewers inspect when quality drifts?
	4. How quickly can a bad prompt change be rolled back?
</Steps>
